# Quickstart

Get started with SkuldBot components in minutes. This guide will walk you through creating your first automation workflow using the visual Studio editor. {{ className: 'lead' }}

<Note>
  This documentation covers the **222 RPA components** available in SkuldBot Studio.
  For installation and setup, see the main SkuldBot documentation.
</Note>

## Understanding Components

Every component in SkuldBot has three key parts:

1. **Configuration** - Input fields that control how the component behaves
2. **Output Variables** - Data produced by the component that can be used by other components
3. **Connections** - Success and Error paths that determine the flow

## Your First Workflow

Let's create a simple workflow that reads an Excel file and logs its contents.

### Step 1: Add a Trigger

Every workflow starts with a trigger. Drag a **Manual Trigger** from the Triggers category:

```json
{
  "id": "trigger-1",
  "type": "trigger.manual",
  "config": {},
  "outputs": {
    "success": "read-excel-1"
  }
}
```

### Step 2: Read Excel Data

Add an **Excel Read Range** component to read data from a spreadsheet:

```json
{
  "id": "read-excel-1",
  "type": "excel.read_range",
  "config": {
    "file": "/path/to/data.xlsx",
    "sheet": "Sheet1",
    "range": "A1:D100"
  },
  "outputs": {
    "success": "log-1",
    "error": "error-handler"
  }
}
```

### Step 3: Log the Results

Add a **Log Message** component to see the data:

```json
{
  "id": "log-1",
  "type": "logging.log_message",
  "config": {
    "level": "info",
    "message": "Read ${Read Excel.rowCount} rows from Excel"
  },
  "outputs": {
    "success": null
  }
}
```

## Using Variables

Components produce output variables that you can reference in subsequent components using the syntax `${Node Label.variableName}`.

For example, after the Excel Read Range component executes, you can access:

- `${Read Excel.data}` - The data as a list of dictionaries
- `${Read Excel.rowCount}` - Number of rows read
- `${Read Excel.columns}` - Column names

## Error Handling

Every component has two output paths:

- **Success** (green line) - Executed when the component completes successfully
- **Error** (orange line) - Executed when the component fails

When connected via the error path, you have access to special error variables:

- `${LAST_ERROR}` - Error message
- `${LAST_ERROR_NODE}` - ID of the node that failed
- `${LAST_ERROR_TYPE}` - Type of the failed node
