<NodeHeader id="trigger-file_watch" icon="FolderSearch" label="File Watch" description="Trigger when file changes" bgColor="#dcfce7" textColor="#166534" />

<Properties>
  <Property name="type" type="string">
    `trigger.file_watch`
  </Property>
</Properties>

### When to Use

- Process invoices dropped into a folder
- Monitor log files for errors
- Automated document ingestion
- Sync files from external sources

### Configuration

- **path** (`text`, required) — Watch path (e.g., `/path/to/folder`)
- **pattern** (`text`) — File pattern (e.g., `*.csv`, `*.pdf`)

### Output

- **filePath** (`string`) — Full path of the changed file
- **fileName** (`string`) — Name of the changed file
- **event** (`string`) — Event type: `created`, `modified`, `deleted`
- **timestamp** (`string`) — When the event occurred

## Choosing the Right Trigger

| Use Case | Recommended Trigger |
|----------|---------------------|
| User-initiated actions | `trigger.manual` or `trigger.form` |
| File-based workflows | `trigger.file_watch` or `trigger.storage_event` |
| External integrations | `trigger.webhook` |
| Scheduled tasks | Schedule in Orchestrator |
| Email processing | `trigger.email_received` |
| Database reactions | `trigger.database_change` |
| Microservices | `trigger.message_bus` or `trigger.queue` |
| ChatOps | `trigger.chat` |
