Enterprise-grade multi-provider storage abstraction. Connect to AWS S3, Azure Blob, Google Cloud Storage, SharePoint, OneDrive, Google Drive, SFTP, FTP, WebDAV, and more. {{ className: 'lead' }}

<Note variant="warning">
  **Required:** All `files.*` nodes require a Storage Provider connection. Even for local filesystem operations, you must connect a Storage Provider node configured with "Local Filesystem" type.
</Note>

<Note>
  Storage Providers provide a unified interface for file operations across local filesystem and 10+ cloud storage backends. Configure once, use everywhere.
</Note>

## Overview {{ id: 'overview' }}

The Storage Provider system provides a unified interface for file operations across 11 different storage backends. This enables:

- **Unified API**: Same file operations work on any storage provider
- **Cloud Migration**: Move from local storage to cloud without changing workflows
- **Cross-Provider Transfer**: Copy/sync files between different providers
- **Enterprise Security**: Encryption, audit logging, checksum verification

### Supported Providers

| Provider | Type | Use Case |
|----------|------|----------|
| **Local** | Filesystem | Development, on-premise |
| **AWS S3** | Cloud | AWS workloads, static files |
| **MinIO** | S3-Compatible | Self-hosted S3 |
| **Azure Blob** | Cloud | Azure workloads, data lakes |
| **Google Cloud Storage** | Cloud | GCP workloads |
| **SharePoint** | Microsoft 365 | Document libraries, team sites |
| **OneDrive** | Microsoft 365 | Personal/business files |
| **Google Drive** | Cloud | Personal/team drives |
| **SFTP** | Protocol | Secure file transfer |
| **FTP/FTPS** | Protocol | Legacy systems |
| **WebDAV** | Protocol | NextCloud, ownCloud |

### How It Works

The Storage Provider node configures a storage backend that can be used by any `files.*` node. Connect it to file operations to switch from local filesystem to cloud storage.

<FlowDiagram items={[
  { label: "storage.provider", sublabel: "AWS S3", type: "primary" },
  { label: "Configuration", sublabel: "bucket, region, credentials", type: "default" },
  { label: "Options", sublabel: "audit, checksums, encryption", type: "success" }
]} />

### Configuration

<Properties>
  <Property name="provider" type="select" required>
    **Provider** — Storage backend type: `local`, `s3`, `minio`, `azure_blob`, `gcs`, `sharepoint`, `onedrive`, `google_drive`, `sftp`, `ftp`, `webdav`.
  </Property>
  <Property name="enable_audit" type="boolean">
    **Enable Audit Logging** — Log all file operations for compliance. Default: `true`.
  </Property>
  <Property name="enable_checksums" type="boolean">
    **Verify Checksums** — SHA-256 integrity verification. Default: `true`.
  </Property>
  <Property name="chunk_size_mb" type="number">
    **Chunk Size** — For streaming large files. Default: `8` MB.
  </Property>
</Properties>

### Output

```json
{
  "provider": "s3",
  "connected": true,
  "endpoint": "s3://my-bucket"
}
```

<NodeHeader id="provider-s3" icon="Cloud" label="AWS S3" description="Amazon S3 and S3-compatible storage" bgColor="#fff7ed" textColor="#ea580c" />

### Configuration

<Properties>
  <Property name="bucket" type="text" required>
    **Bucket Name** — S3 bucket name.
  </Property>
  <Property name="region" type="select">
    **Region** — AWS region (`us-east-1`, `eu-west-1`, etc.).
  </Property>
  <Property name="endpoint_url" type="text">
    **Custom Endpoint** — For S3-compatible services (MinIO, DigitalOcean Spaces).
  </Property>
  <Property name="access_key" type="text">
    **Access Key ID** — AWS access key. Use `${vault.aws_access_key}`.
  </Property>
  <Property name="secret_key" type="password">
    **Secret Access Key** — AWS secret key. Use `${vault.aws_secret_key}`.
  </Property>
  <Property name="encryption" type="select">
    **Server-Side Encryption** — `SSE-S3` (AES-256), `SSE-KMS` (AWS KMS).
  </Property>
  <Property name="kms_key_id" type="text">
    **KMS Key ID** — For SSE-KMS encryption.
  </Property>
</Properties>

### Features

- ✅ Multipart upload for large files (>100MB)
- ✅ Server-side encryption (SSE-S3, SSE-KMS)
- ✅ Presigned URLs for temporary access
- ✅ Transfer acceleration
- ✅ Versioning support

### Example

```json
{
  "provider": "s3",
  "bucket": "my-data-bucket",
  "region": "us-east-1",
  "access_key": "${vault.aws_access_key}",
  "secret_key": "${vault.aws_secret_key}",
  "encryption": "SSE-KMS",
  "kms_key_id": "arn:aws:kms:us-east-1:123456789:key/abc123"
}
```

### Python Dependencies

Install the required packages for each provider:

```bash
# AWS S3 / MinIO
pip install boto3

# Azure Blob Storage
pip install azure-storage-blob azure-identity

# Google Cloud Storage
pip install google-cloud-storage

# Google Drive
pip install google-api-python-client google-auth

# SFTP
pip install paramiko

# SharePoint/OneDrive/WebDAV (only requests needed)
pip install requests
```
