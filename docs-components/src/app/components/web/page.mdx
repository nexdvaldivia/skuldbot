## Selector Types

All web components that interact with elements support multiple selector strategies:

| Type | Description | Example |
|------|-------------|---------|
| **CSS Selector** | Standard CSS selectors | `#submit-btn`, `.form-input`, `div[data-id="123"]` |
| **XPath** | XML Path expressions for complex queries | `//button[@type="submit"]`, `//div[contains(@class,"item")]` |
| **ID** | Element ID attribute | `login-button` |
| **Name** | Element name attribute | `username` |
| **Class** | CSS class name | `btn-primary` |

<Collapsible title="When to use XPath vs CSS">

**Use XPath when:**
- You need to select elements by text content: `//button[text()="Submit"]`
- You need parent/ancestor traversal: `//span[@class="price"]/ancestor::div`
- You need complex conditions: `//input[@type="text" and @required]`
- Working with tables: `//table//tr[3]/td[2]`

**Use CSS when:**
- You have unique IDs or classes
- You need better performance (CSS is faster)
- You're familiar with CSS selectors from web development

</Collapsible>

### Setup Guide


<Collapsible title="Browser Options">

**Headless Mode:** Set `headless: true` to run without visible window (faster, ideal for servers)

**Proxy:** Configure `proxy.server`, `proxy.username`, and `proxy.password` using vault secrets

**Custom User Agent:** Set `userAgent` to a custom browser identification string

</Collapsible>

## XPath Quick Reference

Common XPath patterns for web automation:

### Basic Selection

| XPath | Description |
|-------|-------------|
| `//div` | All div elements |
| `//div[@id='main']` | Div with id="main" |
| `//div[@class='item']` | Div with class="item" |
| `//input[@type='text']` | Text input fields |
| `//a[@href]` | All links with href |

### Text-Based Selection

| XPath | Description |
|-------|-------------|
| `//button[text()='Submit']` | Button with exact text |
| `//span[contains(text(),'Price')]` | Element containing text |
| `//label[starts-with(text(),'Name')]` | Element starting with text |
| `//*[normalize-space()='Login']` | Element with normalized text |

### Attribute Conditions

| XPath | Description |
|-------|-------------|
| `//div[contains(@class,'active')]` | Class contains "active" |
| `//input[@name='email']` | Input by name |
| `//div[@data-testid='modal']` | By data attribute |
| `//a[contains(@href,'download')]` | Link containing "download" |

### Position & Index

| XPath | Description |
|-------|-------------|
| `//ul/li[1]` | First li element |
| `//ul/li[last()]` | Last li element |
| `//table//tr[2]/td[3]` | Row 2, Column 3 |
| `(//div[@class='item'])[1]` | First matching div |

### Traversal

| XPath | Description |
|-------|-------------|
| `//span[@class='price']/..` | Parent of element |
| `//div[@id='form']//input` | Descendant inputs |
| `//label/following-sibling::input` | Input after label |
| `//td/preceding-sibling::td[1]` | Previous cell |

### Multiple Conditions

| XPath | Description |
|-------|-------------|
| `//input[@type='text' and @required]` | Text AND required |
| `//button[@type='submit' or @class='primary']` | Submit OR primary |
| `//div[not(@hidden)]` | Not hidden divs |

<Note>
  **Tip:** Use browser DevTools (F12) → Elements → Right-click element → "Copy XPath" to get the XPath of any element.
</Note>
