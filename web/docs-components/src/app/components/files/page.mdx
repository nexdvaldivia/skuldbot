## Storage Provider Required

**All file operations require a Storage Provider connection.** This ensures consistent behavior whether you're working with local files, AWS S3, Azure Blob, or any other storage backend.

<Note>
**Local files?** Use `storage.provider` with type "Local" and set the base path.

**Cloud files?** Use `storage.provider` with S3, Azure, GCS, etc. and configure credentials.
</Note>

See the [Storage Provider](/components/storage) documentation for configuration details.

<FlowDiagram items={[

  { label: "files.read", sublabel: "Path: /data/config.json", type: "warning" },
  { label: "Output", sublabel: "content, size, provider", type: "success" }

### Configuration

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `path` | text | Yes | File path (relative to provider's base path) |
| `encoding` | select | No | `utf-8` (default), `ascii`, `latin-1`, `binary` |

### Output

```json
{
  "content": "File contents here...",
  "path": "/data/report.txt",
  "size": 1024,
  "provider": "my-s3-bucket"
}
```

<NodeHeader id="files-write" icon="FileOutput" label="Write File" description="Writes content to a file in any storage provider" bgColor="#fef3c7" textColor="#b45309" />

### Configuration

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `path` | text | Yes | Destination file path |
| `content` | textarea | Yes | Text to write. Supports expressions: `${prev.data}` |
| `append` | boolean | No | Append to existing file (default: false) |
| `content_type` | text | No | MIME type (auto-detected if not specified) |

### Output

```json
{
  "path": "/output/report.csv",
  "size": 2048,
  "status": "success"
}
```

<NodeHeader id="files-watch" icon="Eye" label="Watch Folder" description="Monitors a folder for file changes (local only)" bgColor="#fef3c7" textColor="#b45309" />

<Note>
**Local Only:** `files.watch` only works with local filesystem. Cloud storage providers (S3, Azure, etc.) do not support real-time file watching.
</Note>

### Configuration

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `path` | text | Yes | Directory to watch |
| `pattern` | text | No | File pattern (e.g., `*.csv`) |
| `timeout` | number | No | Max wait time in seconds (default: 60) |

### Output

```json
{
  "changed_file": "/inbox/new_invoice.pdf",
  "event": "created",
  "timestamp": "2024-01-15T10:30:00Z",
  "timeout_reached": false
}
```

Events: `created`, `modified`, `deleted`, `none`

<NodeHeader id="files-presigned_url" icon="Link" label="Presigned URL" description="Generates a temporary access URL for a file (cloud only)" bgColor="#fef3c7" textColor="#b45309" />

<Note>
**Cloud Only:** `files.presigned_url` only works with cloud storage providers (S3, Azure Blob, GCS). Local filesystem does not support presigned URLs.
</Note>

### Configuration

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `path` | text | Yes | File path in cloud storage |
| `expiration` | number | No | URL validity in seconds (default: 3600) |

### Output

```json
{
  "url": "https://bucket.s3.amazonaws.com/file.pdf?X-Amz-Signature=...",
  "expires_at": "2024-01-15T11:30:00Z"
}
```

## Cross-Provider Operations

For operations between different storage providers, use the dedicated transfer nodes:

| Operation | Node | Description |
|-----------|------|-------------|
| Copy between providers | [storage.transfer](/components/storage#storage-transfer) | Transfer file from one provider to another |
| Sync directories | [storage.sync](/components/storage#storage-sync) | Sync entire directories between providers |
