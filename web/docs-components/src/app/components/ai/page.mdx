<Note>
  The AI Agent is the most powerful component in SkuldBot. It can **reason**, **decide**, and **execute** any combination of the 280+ available tools autonomously. Unlike fixed workflows, the agent adapts its behavior based on context.
</Note>

### How It Works

The agent follows a **ReAct loop** (Reason + Act):

1. **Observe** — Read the current state and goal
2. **Reason** — Decide what to do next
3. **Act** — Call a tool (any connected node)
4. **Observe** — Read the tool result
5. **Repeat** — Until the goal is achieved or max iterations reached

### Configuration

<Properties>
  <Property name="name" type="text">
    Agent name. Used in logs and Evidence Pack.
  </Property>
  <Property name="goal" type="textarea" required>
    Goal / Task. Describe what the agent should accomplish. Be specific — the more context you give, the better the agent performs.
  </Property>
  <Property name="system_prompt" type="textarea">
    System prompt that defines the agent's behavior and constraints. Default: "You are a helpful assistant that can use tools to accomplish tasks."
  </Property>
  <Property name="max_iterations" type="number">
    Maximum reasoning iterations before stopping. Default: `10`. Increase for complex multi-step tasks.
  </Property>
</Properties>

### Connections

The AI Agent has **4 input handles** at the bottom:

| Handle | Color | Connects To | Required |
|--------|-------|-------------|:--------:|
| **Model** | Blue | `ai.model` node | Yes |
| **Embed** | — | `ai.embeddings` node | No |
| **Memory** | — | `vectordb.memory` node | No |
| **Tools** | Violet | Any other node (top handle) | Yes (at least 1) |

<Collapsible title="What can be a tool?">

Almost **every node in SkuldBot** can be used as a tool for the agent. Each node has a violet square handle at the top — drag from it to the agent's Tools handle.

Available as tools:
- **Web**: Open browser, navigate, click, type, screenshot, extract text
- **Excel**: Create, read, write, format, chart, filter, vlookup
- **Email**: Send, read, reply, search, download attachments
- **Files**: Read, write, copy, move, delete, list, zip
- **Database**: Query, insert, update, delete, transactions
- **API**: HTTP GET/POST/PUT/DELETE, GraphQL, SOAP
- **Documents**: PDF read/write/merge, OCR, Word, HTML to PDF
- **Desktop**: Open apps, click, type, hotkeys, screenshots
- **Code**: Run JavaScript or Python
- **AI tasks**: Extract data, classify, summarize, translate, sentiment
- **Human**: Request approval, input, review
- **Data**: Singer taps/targets (23+ connectors)
- **Voice**: Call, speak, listen, transfer
- **Storage**: Upload, download, sync across providers

**Not available** as tools (they are configuration nodes):
- Triggers (they start the flow, not tools)
- AI Model / Embeddings (configuration, connect to Model/Embed handles instead)
- Vector Memory (connect to Memory handle instead)

</Collapsible>

### Example: Invoice Processing Agent

```json
{
  "nodes": [
    {
      "id": "model-1",
      "type": "ai.model",
      "config": {
        "provider": "openai",
        "model": "gpt-4o",
        "api_key": "${vault.OPENAI_API_KEY}",
        "temperature": 0.1
      }
    },
    {
      "id": "agent-1",
      "type": "ai.agent",
      "config": {
        "name": "Invoice Processor",
        "goal": "Read the PDF invoice, extract the vendor name, invoice number, date, line items, and total amount. Write the data to the Excel spreadsheet.",
        "system_prompt": "You are a precise data extraction agent. Always verify extracted data before writing.",
        "max_iterations": 10
      }
    },
    {
      "id": "tool-pdf",
      "type": "document.pdf_read",
      "config": { "file_path": "${INVOICE_PATH}" }
    },
    {
      "id": "tool-extract",
      "type": "ai.extract_data",
      "config": {
        "schema": "{\"vendor\": \"string\", \"invoice_number\": \"string\", \"date\": \"string\", \"line_items\": \"array\", \"total\": \"number\"}"
      }
    },
    {
      "id": "tool-excel",
      "type": "excel.write_range",
      "config": {
        "file_path": "invoices.xlsx",
        "sheet": "Sheet1",
        "range": "A1"
      }
    }
  ]
}
```

In this example, the agent:
1. Uses `document.pdf_read` to read the invoice PDF
2. Uses `ai.extract_data` to extract structured fields
3. Uses `excel.write_range` to write the data to a spreadsheet
4. Decides the order and handles errors automatically

### Example: Customer Support Agent with Memory

```json
{
  "nodes": [
    {
      "id": "model-1",
      "type": "ai.model",
      "config": { "provider": "anthropic", "model": "claude-3-5-sonnet-20241022" }
    },
    {
      "id": "embeddings-1",
      "type": "ai.embeddings",
      "config": { "provider": "openai", "model": "text-embedding-3-small" }
    },
    {
      "id": "memory-1",
      "type": "vectordb.memory",
      "config": {
        "provider": "chroma",
        "collection": "support_knowledge",
        "memory_type": "both",
        "top_k": 5
      }
    },
    {
      "id": "agent-1",
      "type": "ai.agent",
      "config": {
        "name": "Support Agent",
        "goal": "Answer the customer question using the knowledge base. If you can't find the answer, search emails for relevant information.",
        "max_iterations": 8
      }
    },
    {
      "id": "tool-search-email",
      "type": "email.search",
      "config": { "folder": "Support" }
    },
    {
      "id": "tool-reply",
      "type": "email.send",
      "config": {}
    }
  ]
}
```

The agent automatically retrieves relevant context from the vector memory (RAG) before responding, and stores important interactions for future reference.

### Best Practices

<Collapsible title="Writing effective goals">

**Good goals** are specific and measurable:
- "Read the Excel file at /data/claims.xlsx, find all claims with status 'pending' older than 30 days, and send a summary email to claims@company.com"
- "Extract the patient name, DOB, and diagnosis from the medical record PDF, then create a FHIR resource"

**Bad goals** are vague:
- "Process the file" (which file? process how?)
- "Handle the data" (what data? what does "handle" mean?)

</Collapsible>

<Collapsible title="How many tools to connect">

- **3-5 tools**: Best for focused tasks. Agent is fast and reliable.
- **5-10 tools**: Good for complex workflows. Agent may need more iterations.
- **10+ tools**: The agent can handle it, but increase `max_iterations` to 15-20 and be very specific in the goal.

The agent only calls tools that are relevant. Connecting 20 tools doesn't mean it will use all 20 — it picks what it needs.

</Collapsible>

<Collapsible title="Agent vs Fixed Flow — when to use which">

| Use **Fixed Flow** when | Use **AI Agent** when |
|--------------------------|----------------------|
| Steps are always the same | Steps vary based on content |
| Speed is critical | Accuracy/reasoning is critical |
| No decision-making needed | Decisions depend on data |
| Simple A→B→C pipeline | Complex multi-tool orchestration |
| Compliance requires exact steps | Goal-oriented, flexible execution |

You can **combine both**: use a fixed flow for the main pipeline and embed an AI Agent node for the parts that require reasoning.

</Collapsible>

### Evidence Pack Integration

Every agent execution generates a complete audit trail in the Evidence Pack:
- All reasoning steps (thought, action, observation)
- Every tool call with inputs and outputs
- LLM prompts and responses (redacted if containing PII/PHI)
- Token usage and latency per call
- Confidence scores for decisions
