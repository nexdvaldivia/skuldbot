## Nodes Reference

### bot.call

<NodeHeader id="call-bot" icon="Package" label="Call Bot" description="Invoke another bot as a subprocess, passing parameters and receiving results" bgColor="#fce7f3" textColor="#be185d" />

**Configuration:**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `bot_id` | bot-select | Yes | Bot to call from this project |
| `parameters` | JSON | No | Parameters to pass (supports expressions) |
| `wait_for_completion` | boolean | No | Wait for result (default: true) |
| `timeout_seconds` | number | No | Max wait time (default: 300) |
| `retry_count` | number | No | Retry attempts on failure |
| `retry_delay_seconds` | number | No | Delay between retries |
| `fail_on_error` | boolean | No | Stop parent if subbot fails |
| `inherit_context` | boolean | No | Pass vault/env to subbot |

**Output:**

```json
{
  "result": { /* returned by bot.output */ },
  "success": true,
  "error": null,
  "execution_id": "exec_a1b2c3d4",
  "duration_ms": 1234,
  "retry_attempts": 0
}
```

**Example - Sync Call:**

```json
{
  "type": "bot.call",
  "config": {
    "bot_id": "validate-email-bot",
    "parameters": {
      "email": "${data.customerEmail}",
      "strict": true
    },
    "timeout_seconds": 30,
    "fail_on_error": true
  }
}
```

**Example - Async (Fire & Forget):**

```json
{
  "type": "bot.call",
  "config": {
    "bot_id": "send-notification-bot",
    "parameters": {
      "message": "Process completed",
      "channel": "slack"
    },
    "wait_for_completion": false
  }
}
```

### bot.input

<NodeHeader id="bot-input" icon="LogIn" label="Bot Input" description="Define input parameters for a bot when called as a subprocess" bgColor="#fce7f3" textColor="#be185d" />

Use this **instead of a trigger** for bots designed to be called by other bots.

**Configuration:**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `parameters` | JSON | Yes | Parameter schema definition |
| `validate_inputs` | boolean | No | Validate types/required (default: true) |
| `fail_on_invalid` | boolean | No | Fail if validation fails (default: true) |

**Parameter Schema Format:**

```json
[
  {
    "name": "email",
    "type": "string",
    "required": true,
    "description": "Email address to validate"
  },
  {
    "name": "items",
    "type": "array",
    "required": false,
    "default": []
  },
  {
    "name": "maxRetries",
    "type": "number",
    "required": false,
    "default": 3
  }
]
```

**Output:**

```json
{
  "params": {
    "email": "user@example.com",
    "items": [],
    "maxRetries": 3
  },
  "caller_bot_id": "main-workflow-bot",
  "caller_execution_id": "exec_xyz789",
  "validation_passed": true,
  "validation_errors": []
}
```

### bot.output

<NodeHeader id="bot-output" icon="LogOut" label="Bot Output" description="Define what a subprocess bot returns to its caller" bgColor="#fce7f3" textColor="#be185d" />

The return value is available in the calling bot's `bot.call` output as `${CallBot.result}`.

**Configuration:**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `return_value` | JSON | Yes | Value to return (supports expressions) |
| `status` | select | No | Exit status: success, error, warning |
| `message` | text | No | Human-readable status message |

**Example:**

```json
{
  "type": "bot.output",
  "config": {
    "return_value": {
      "valid": "${validation.passed}",
      "score": "${validation.score}",
      "errors": "${validation.errors}",
      "processedAt": "${NOW}"
    },
    "status": "success",
    "message": "Validation completed: ${validation.score}%"
  }
}
```

### bot.queue

<NodeHeader id="bot-queue" icon="ListTodo" label="Queue Bot Calls" description="Execute multiple bot calls in parallel with concurrency control" bgColor="#fce7f3" textColor="#be185d" />

Perfect for batch processing scenarios.

**Configuration:**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `bot_id` | bot-select | Yes | Bot to call for each item |
| `items` | expression | Yes | Array of items to process |
| `parameter_mapping` | JSON | Yes | Map item fields to bot parameters |
| `concurrency` | number | No | Max parallel executions (default: 3) |
| `fail_fast` | boolean | No | Stop all on first failure |
| `collect_results` | boolean | No | Gather results into array |
| `timeout_per_item` | number | No | Timeout per item (seconds) |

**Parameter Mapping Example:**

```json
{
  "invoice_id": "${item.id}",
  "amount": "${item.total}",
  "customer": "${item.customer.name}"
}
```

**Output:**

```json
{
  "results": [
    { "valid": true, "invoice_id": "INV-001" },
    { "valid": true, "invoice_id": "INV-002" },
    { "valid": false, "invoice_id": "INV-003", "error": "Invalid amount" }
  ],
  "total": 3,
  "succeeded": 2,
  "failed": 1,
  "errors": [
    { "index": 2, "error": "Invalid amount" }
  ],
  "duration_ms": 4521
}
```

## Enterprise Features

| Feature | Description |
|---------|-------------|
| **Timeout Control** | Configurable per-call timeout prevents runaway subbots |
| **Retry with Backoff** | Automatic retry with exponential backoff on failures |
| **Fail-Fast Mode** | Stop batch processing on first error |
| **Context Inheritance** | Pass vault secrets and env vars to subbots |
| **Execution Tracking** | Unique execution IDs for audit trails |
| **Async Execution** | Fire-and-forget for non-blocking operations |
| **Concurrency Control** | Limit parallel executions to prevent resource exhaustion |
| **Parameter Validation** | Schema-based input validation with type checking |

## Best Practices

1. **Keep Subbots Focused**: Each subbot should do one thing well
2. **Define Clear Contracts**: Use `bot.input` schema to document expected parameters
3. **Handle Errors Gracefully**: Use `fail_on_error: false` when appropriate
4. **Set Reasonable Timeouts**: Prevent hung processes from blocking workflows
5. **Use Async for Notifications**: Fire-and-forget for non-critical side effects
6. **Limit Concurrency**: Match concurrency to available resources
7. **Log Meaningful Messages**: Use `bot.output` message for debugging
