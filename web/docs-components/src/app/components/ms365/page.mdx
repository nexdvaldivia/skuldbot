# Microsoft 365

Deep Microsoft 365 integration via Microsoft Graph API. Outlook email, Calendar, OneDrive, Teams, and SharePoint. {{ className: 'lead' }}

<Note>
  **20 components** for email operations plus folder management. Requires Azure AD app registration. For basic email (SMTP/IMAP), see [Email](/components/email) instead. Attachment download/send nodes support [Storage Provider](/components/storage) for cloud file access.
</Note>

### When to Use MS365 vs Email

| Use **MS365** when | Use **Email** when |
|---------------------|-------------------|
| Your org uses Microsoft 365 / Exchange Online | You use Gmail, generic SMTP, or IMAP |
| You need categories, flags, importance | You just need send/read/reply |
| You need OData filters and Microsoft Search | Basic subject/sender filters are enough |
| You need folder management | No folder operations needed |
| You need Graph API features (Calendar, Teams later) | Only email operations |

### Azure AD Setup

To use MS365 components, register an app in Azure AD:

1. Go to **Azure Portal** > **Azure Active Directory** > **App registrations** > **New registration**
2. Name: "SkuldBot Automation" (or your preference)
3. Supported account type: **Single tenant** (your org only)
4. After registration, note the **Application (client) ID** and **Directory (tenant) ID**
5. Go to **Certificates & secrets** > **New client secret** > copy the value
6. Go to **API permissions** > **Add permission** > **Microsoft Graph** > **Application permissions**
7. Add: `Mail.Read`, `Mail.ReadWrite`, `Mail.Send`
8. Click **Grant admin consent**
9. Store credentials in SkuldBot Vault:
   - Tenant ID
   - Client ID
   - Client Secret

## Required Graph API Permissions

| Operation | Permission | Type |
|-----------|------------|------|
| Read emails | `Mail.Read` | Application |
| Send emails | `Mail.Send` | Application |
| Manage emails (move, delete, flag) | `Mail.ReadWrite` | Application |
| Calendar access | `Calendars.ReadWrite` | Application |
| OneDrive files | `Files.ReadWrite` | Application |
| Teams messaging | `ChannelMessage.Send` | Application |

<Note>
  Use **Application permissions** (not Delegated) for bot automation. Application permissions allow the bot to act without a signed-in user, which is required for scheduled/unattended execution.
</Note>

### OData Filter Examples

Use OData filters in `ms365.email_list` for precise queries:

| Filter | Description |
|--------|-------------|
| `isRead eq false` | Unread emails only |
| `hasAttachments eq true` | Emails with attachments |
| `importance eq 'high'` | High importance only |
| `from/emailAddress/address eq 'ceo@company.com'` | From specific sender |
| `receivedDateTime ge 2025-01-01T00:00:00Z` | After a specific date |
| `subject eq 'Invoice'` | Exact subject match |
| `contains(subject, 'urgent')` | Subject contains word |

Combine with `and`: `isRead eq false and hasAttachments eq true and importance eq 'high'`
