# Email

Complete email automation. Send, read, reply, forward emails via SMTP, IMAP, Outlook, or Gmail API. {{ className: 'lead' }}

<Note>
  **15 components**. For Microsoft 365 (Exchange Online) with calendar, teams, and SharePoint, see the [MS365](/components/ms365) category. Download Attachment and Send (with attachments) support [Storage Provider](/components/storage) for cloud file access.
</Note>

---

## Provider Setup {{ id: 'email-provider-setup' }}

SkuldBot supports multiple email providers. Choose based on your infrastructure:

| Provider | Best For | Authentication |
|----------|----------|----------------|
| **SMTP** | Any email server, on-premise | Username + password |
| **IMAP** | Reading from any mailbox | Username + password |
| **Gmail API** | Google Workspace | OAuth2 / Service Account |
| **Outlook** | Microsoft 365 | OAuth2 / App Registration |

<Note>
  **Security**: Never hardcode email credentials. Use the Vault: Vault variables for credentials. Store credentials in the Studio Vault Manager or the Orchestrator credentials store.
</Note>

<NodeHeader id="email-send" icon="Send" label="Send Email" description="Send an email via the default provider" bgColor="#fef3c7" textColor="#92400e" />

<Properties>
  <Property name="type" type="string">
    `email.send`
  </Property>
</Properties>

### Configuration

<Properties>
  <Property name="to" type="text" required>
    Recipient email address(es). Comma-separated for multiple: `alice@co.com, bob@co.com`
  </Property>
  <Property name="subject" type="text" required>
    Email subject line. Supports expressions: `Invoice ${Invoice Number.output}`
  </Property>
  <Property name="body" type="textarea" required>
    Email body content. Can include HTML if `html` is enabled.
  </Property>
  <Property name="cc" type="text">
    CC recipients (comma-separated).
  </Property>
  <Property name="bcc" type="text">
    BCC recipients (comma-separated).
  </Property>
  <Property name="html" type="boolean">
    Send as HTML email. Default: `false`.
  </Property>
  <Property name="attachments" type="text">
    File paths to attach (comma-separated). E.g., `${FILE_PATH}, /tmp/report.pdf`
  </Property>
</Properties>

### Example: Send a report email

```json
{
  "id": "send-report",
  "type": "email.send",
  "config": {
    "to": "manager@company.com",
    "subject": "Daily Sales Report - ${BOT_NAME}",
    "body": "Please find the daily sales report attached.\n\nTotal sales: ${Excel Read.output.rowCount} records processed.",
    "attachments": "${Excel Export.output.filePath}",
    "html": false
  }
}
```

### Output

<Properties>
  <Property name="message_id" type="string">Unique message identifier.</Property>
  <Property name="status" type="string">Send status: `sent` or `failed`.</Property>
</Properties>

<NodeHeader id="email-read" icon="MailOpen" label="Read Emails" description="Read emails from a mailbox folder" bgColor="#fef3c7" textColor="#92400e" />

<Properties>
  <Property name="type" type="string">
    `email.read`
  </Property>
</Properties>

### Configuration

<Properties>
  <Property name="folder" type="text">
    Mailbox folder. Default: `INBOX`. Common: `INBOX`, `Sent`, `Drafts`, `Archive`.
  </Property>
  <Property name="filter" type="text">
    Subject filter (partial match). E.g., `Invoice` matches "Invoice #123".
  </Property>
  <Property name="unread_only" type="boolean">
    Only fetch unread emails. Default: `true`.
  </Property>
  <Property name="limit" type="number">
    Maximum emails to retrieve. Default: `10`.
  </Property>
  <Property name="since" type="text">
    Only emails after this date. Format: `YYYY-MM-DD`.
  </Property>
</Properties>

### Example: Read unread invoices

```json
{
  "id": "read-invoices",
  "type": "email.read",
  "config": {
    "folder": "INBOX",
    "filter": "Invoice",
    "unread_only": true,
    "limit": 50,
    "since": "2025-01-01"
  }
}
```

### Output

<Properties>
  <Property name="emails" type="array">
    List of email objects, each containing:
    - `message_id` — Unique message ID
    - `from` — Sender address
    - `to` — Recipient(s)
    - `subject` — Subject line
    - `body` — Plain text body
    - `html_body` — HTML body (if available)
    - `date` — Received date
    - `attachments` — List of attachment names
    - `is_read` — Read status
  </Property>
  <Property name="count" type="number">Number of emails retrieved.</Property>
</Properties>

<NodeHeader id="email-send_smtp" icon="Server" label="Send via SMTP" description="Send email through a specific SMTP server" bgColor="#fef3c7" textColor="#92400e" />

<Properties>
  <Property name="type" type="string">
    `email.send_smtp`
  </Property>
</Properties>

### Configuration

<Properties>
  <Property name="smtp_host" type="text" required>
    SMTP server address. E.g., `smtp.gmail.com`, `smtp.office365.com`, `mail.company.com`.
  </Property>
  <Property name="port" type="number">
    SMTP port. Default: `587`. Common: `587` (TLS), `465` (SSL), `25` (unencrypted).
  </Property>
  <Property name="username" type="text" required>
    SMTP username. Use Vault for secure storage.
  </Property>
  <Property name="password" type="password" required>
    SMTP password. **Always use Vault** for passwords.
  </Property>
  <Property name="tls" type="boolean">
    Use TLS encryption. Default: `true`. **Never disable in production.**
  </Property>
  <Property name="to" type="text" required>Recipient.</Property>
  <Property name="from" type="text">Sender address (if different from username).</Property>
  <Property name="subject" type="text" required>Subject.</Property>
  <Property name="body" type="textarea" required>Body.</Property>
</Properties>

### Example: Send via corporate SMTP

```json
{
  "id": "smtp-send",
  "type": "email.send_smtp",
  "config": {
    "smtp_host": "smtp.company.com",
    "port": 587,
    "username": "${vault.SMTP_USER}",
    "password": "${vault.SMTP_PASSWORD}",
    "tls": true,
    "to": "client@external.com",
    "from": "automation@company.com",
    "subject": "Automated Report",
    "body": "Please find the attached report."
  }
}
```

<Note>
  For Gmail SMTP, enable "Less secure app access" or use an App Password. For Outlook, use `smtp.office365.com:587`.
</Note>
