Build RAG (Retrieval-Augmented Generation) applications. Store embeddings, perform semantic search, and provide long-term memory for AI agents. {{ className: 'lead' }}

<Note>
  Supported vector databases: **Pinecone**, **Chroma**, **Qdrant**, **Weaviate**, **Milvus**, and local file-based storage. Load Documents node supports [Storage Provider](/components/storage) for ingesting documents from any cloud storage.
</Note>

## What is RAG?

RAG enhances AI responses by retrieving relevant context from a knowledge base before generating answers.

  </Property>

  </Property>
  <Property name="dimensions" type="number">
    **Dimensions** — Vector dimensions. Auto-detected if empty.
  </Property>
</Properties>

<NodeHeader id="vectordb-upsert" icon="Upload" label="Upsert" description="Store vectors in database" bgColor="#fce7f3" textColor="#be185d" />

<Properties>
  <Property name="type" type="string">
    `vectordb.upsert`
  </Property>
</Properties>

### Configuration

<Properties>
  <Property name="documents" type="textarea" required>
    **Documents** — Array of documents to store. Each needs `text` and optional `metadata`.
  </Property>
  <Property name="collection" type="text" required>
    **Collection** — Target collection/index name.
  </Property>
</Properties>

### Example

```json
{
  "documents": "[{\"id\": \"doc1\", \"text\": \"Our refund policy...\", \"metadata\": {\"source\": \"policy.pdf\", \"page\": 5}}]",
  "collection": "knowledge_base"
}
```

### Output

```json
{
  "upserted": 1,
  "collection": "knowledge_base"
}
```

<NodeHeader id="vectordb-query" icon="Search" label="Query" description="Semantic search in vector database" bgColor="#fce7f3" textColor="#be185d" />

<Properties>
  <Property name="type" type="string">
    `vectordb.query`
  </Property>
</Properties>

### Configuration

<Properties>
  <Property name="query" type="text" required>
    **Query** — Search query text.
  </Property>
  <Property name="collection" type="text" required>
    **Collection** — Collection to search.
  </Property>
  <Property name="top_k" type="number">
    **Top K** — Number of results to return. Default: 5.
  </Property>
  <Property name="filter" type="textarea">
    **Filter** — Metadata filter as JSON.
  </Property>
</Properties>

### Output

```json
{
  "results": [
    {
      "id": "doc1",
      "text": "Our refund policy allows returns within 30 days...",
      "score": 0.92,
      "metadata": {"source": "policy.pdf", "page": 5}
    },
    {
      "id": "doc2",
      "text": "For refunds, contact customer support...",
      "score": 0.85,
      "metadata": {"source": "faq.md"}
    }
  ]
}
```
