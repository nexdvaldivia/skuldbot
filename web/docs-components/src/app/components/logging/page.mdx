# Logging & Monitoring

Monitor your bots. Logging, screenshots, alerts, and audit trails. {{ className: 'lead' }}

<Note>
  **8 components**. All log entries are included in the Evidence Pack for audit compliance. PII/PHI in log messages is automatically redacted when compliance policies are active. Log Export supports [Storage Provider](/components/storage) for cloud log storage.
</Note>

### Best Practices

- Use `INFO` for normal operations, `WARN` for recoverable issues, `ERROR` for failures
- Use `logging.metric` to track KPIs: records processed, processing time, error rate
- Use `logging.timer_start` / `logging.timer_stop` to measure performance of critical sections
- Use `logging.audit` for compliance-sensitive operations (data access, decisions, approvals)
- Use `logging.notification` to alert teams on failures or thresholds

### Compliance Audit Trail

For regulated workflows, add audit entries at every decision point:

<FlowDiagram items={[
  { label: "ai.classify", sublabel: "Classify claim", type: "primary" },
  { label: "logging.audit", sublabel: "Decision logged", type: "default" },

Every audit entry includes: who, what, when, and the data context. Combined with the Evidence Pack, this provides a complete chain of custody for auditors.
