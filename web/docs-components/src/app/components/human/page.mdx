# Human-in-the-Loop

Add human review to your automations. Approvals, reviews, and escalations for critical decisions. {{ className: 'lead' }}

<Note>
  **5 components**. Human tasks create notifications and **pause the workflow** until the user takes action. This is critical for regulated industries where AI decisions require human oversight.
</Note>

### Why HITL Matters

In enterprise automation, not everything should be fully automatic:

| Scenario | Why HITL |
|----------|----------|
| Claim over $10,000 | Regulatory requirement for supervisor review |
| AI confidence below 80% | Low-confidence AI decisions need human verification |
| PII data export | Compliance requires explicit approval before sending PII externally |
| Bot encounters unknown scenario | Human decides: retry, skip, or abort |
| Financial transactions | Segregation of duties requires approval chain |

### Evidence Pack Integration

Every HITL interaction is recorded in the Evidence Pack:
- Who was notified and when
- What data was presented
- What decision was made
- Who approved/rejected and when
- Time to respond

### Exception Handling with Human Recovery

When a bot encounters an unexpected error, instead of failing silently, escalate to a human:

<FlowDiagram items={[
  { label: "api.http_request", sublabel: "Call external API", type: "primary" },
  { label: "human.exception", sublabel: "Human decides: retry/skip/abort", type: "warning" },
  { label: "human.notification", sublabel: "Alert team", type: "success" }
]} />

Connect the **error** (orange) output of any node to `human.exception`. The human sees the error details and chooses: retry the operation, skip it and continue, or abort the workflow.
