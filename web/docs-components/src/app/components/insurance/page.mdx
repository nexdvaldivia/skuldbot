# Insurance

Insurance-specific automation. FNOL processing, policy management, and claims handling. {{ className: 'lead' }}

<Note>
  **4 components** designed for P&C insurance workflows. These integrate with common policy admin systems and produce audit-ready output for regulatory compliance. Combine with [AI](/components/ai) nodes for intelligent claim extraction and with [Compliance](/components/compliance) for PII/PHI protection.
</Note>

<Warning>
  Insurance workflows process sensitive policyholder data (PII). Always include `compliance.protect_pii` nodes and enable Evidence Pack generation. Use Vault for all API credentials. See [Compliance](/components/compliance) for details.
</Warning>

## Example: FNOL Processing Pipeline

A typical First Notice of Loss workflow combines multiple SkuldBot components:

```json
{
  "nodes": [
    {
      "id": "trigger",
      "type": "trigger.voice_inbound",
      "config": { "queue": "fnol-line" }
    },
    {
      "id": "lookup",
      "type": "insurance.lookup_policy",
      "config": {
        "api_url": "https://policy-api.internal/v1/lookup",
        "phone_number": "${trigger.output.callerPhone}",
        "headers": "{\"Authorization\": \"Bearer ${vault.POLICY_API_KEY}\"}"
      }
    },
    {
      "id": "validate",
      "type": "insurance.validate_policy",
      "config": {
        "api_url": "https://policy-api.internal/v1/validate",
        "policy_id": "${lookup.output.policy_id}",
        "claim_type": "auto",
        "check_active": true,
        "check_coverage": true
      }
    },
    {
      "id": "extract",
      "type": "insurance.extract_claim_data",
      "config": {
        "transcript": "${trigger.output.transcript}",
        "claim_type": "auto"
      }
    },
    {
      "id": "protect",
      "type": "compliance.protect_pii",
      "config": { "data": "${extract.output.extracted_data}" }
    },
    {
      "id": "fnol",
      "type": "insurance.fnol_record",
      "config": {
        "caller_phone": "${trigger.output.callerPhone}",
        "caller_name": "${extract.output.extracted_data.caller_name}",
        "policy_number": "${lookup.output.policy_number}",
        "incident_type": "auto",
        "incident_description": "${extract.output.extracted_data.description}",
        "recording_url": "${trigger.output.recordingUrl}",
        "transcript": "${trigger.output.transcript}"
      }
    }
  ]
}
```

This pipeline: receives an inbound call, looks up the caller's policy by phone number, validates coverage, uses AI to extract claim details from the conversation transcript, protects PII, and creates the FNOL record — all with a complete Evidence Pack for audit.

## Compliance Notes

- **Evidence Pack**: Every insurance workflow execution generates an immutable Evidence Pack with full audit trail — timestamps, decisions, data lineage, and screenshots
- **PII Protection**: Policyholder data (name, SSN, DOB, phone) must be redacted in logs. Use `compliance.protect_pii` before any logging or external API call
- **Data Residency**: For US insurance, ensure policy API calls and data storage comply with state-specific regulations
- **Retention**: Insurance records typically require 10+ year retention. Configure Evidence Pack retention accordingly
