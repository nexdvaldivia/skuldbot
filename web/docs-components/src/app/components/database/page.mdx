<NodeHeader id="database-connect" icon="Database" label="Connect Database" description="Connect to database using individual credentials (supports vault references)" bgColor="#fed7aa" textColor="#9a3412" />

<Properties>
  <Property name="type" type="string">
    `database.connect`
  </Property>
</Properties>

**Configuration:**
- **type** (`select`, default: `postgresql`) — Database type:
  - `postgresql` — PostgreSQL
  - `mysql` — MySQL
  - `mssql` — SQL Server
  - `oracle` — Oracle
  - `sqlite` — SQLite
- **host** (`text`, required) — Host. Supports expressions like `${vault.db_host}`
- **port** (`number`, default: `5432`) — Port
- **database** (`text`, required) — Database name
- **username** (`text`, required) — Username. Supports expressions like `${vault.db_user}`
- **password** (`text`, required) — Password. Supports expressions like `${vault.db_password}`
- **ssl** (`boolean`, default: `false`) — Enable SSL

**Output:**
- **connectionId** (`string`) — Connection ID for use in other database nodes
- **connected** (`boolean`) — Whether connection was successful
- **serverVersion** (`string`) — Database server version

**Example:**

```json
{
  "type": "database.connect",
  "config": {
    "type": "postgresql",
    "host": "${VAULT.db_host}",
    "port": 5432,
    "database": "myapp",
    "username": "${VAULT.db_user}",
    "password": "${VAULT.db_password}",
    "ssl": true
  }
}
```

<NodeHeader id="database-query" icon="Code" label="Execute Query" description="Run SQL query on connected database" bgColor="#fed7aa" textColor="#9a3412" />

<Properties>
  <Property name="type" type="string">
    `database.query`
  </Property>
</Properties>

**Configuration:**
- **connection** (`text`) — Connection reference. Supports expressions like `${Connect Database.connectionId}`
- **query** (`textarea`, required) — SQL query. Supports expressions.
- **params** (`textarea`) — Query parameters as JSON. Supports expressions.

**Output:**
- **records** (`array`) — Query result rows
- **columns** (`array`) — Column names
- **recordCount** (`number`) — Number of rows returned

**Example - Parameterized Query:**

```json
{
  "type": "database.query",
  "config": {
    "connection": "${Connect Database.connectionId}",
    "query": "SELECT * FROM customers WHERE status = $1 AND created_at > $2",
    "params": "[\"active\", \"2024-01-01\"]"
  }
}
```

<Warning>
  **Always use parameterized queries** to prevent SQL injection. Never concatenate user input directly into SQL strings.
</Warning>

## Security Best Practices

| Rule | Why |
|------|-----|
| **Always use parameterized queries** | Prevents SQL injection — the #1 database vulnerability |
| **Store credentials in Vault** | Never hardcode host, user, or password in config |
| **Use SSL for remote connections** | Encrypts data in transit |
| **Use read-only credentials for SELECT** | Principle of least privilege |
| **Close connections when done** | Prevents connection pool exhaustion |
| **Use transactions for multi-step writes** | Ensures data consistency on failure |
| **Add audit logging for writes** | Compliance requires tracking who changed what |

## Connection Strings by Database

| Database | Default Port | SSL Note |
|----------|:------------:|----------|
| **PostgreSQL** | 5432 | SSL recommended, use `sslmode=require` |
| **MySQL** | 3306 | SSL via `ssl: true` |
| **SQL Server** | 1433 | Encrypt via `encrypt: true` |
| **Oracle** | 1521 | SSL via Oracle Wallet |
| **SQLite** | N/A | Local file, no network needed |
